# Dota2-OB Fullstack Implementation

Use this skill for code implementation in the Dota2-OB repository.

Implementation standards:

- Read the spec, docs, and existing code before editing.
- Keep changes scoped.
- Prefer typed schemas for GSI/API/replay payloads.
- Persist raw observations before derived analytics.
- Make latency and sampling assumptions explicit.
- Add focused tests and fixtures for parsers, normalizers, and derived metrics.
- Avoid unsafe account, anti-cheat, packet, or memory techniques unless explicitly authorized.
- Never commit secrets.

Completion must include:

- changed files,
- verification commands and results,
- unresolved assumptions,
- review handoff notes.
