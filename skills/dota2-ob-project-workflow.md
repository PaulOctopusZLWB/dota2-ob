# Dota2-OB Project Workflow

Use this skill for any Dota2-OB task that may become implementation work.

Rules:

- Start with a spec unless the task is purely administrative.
- A valid spec includes objective, context, source of truth, requirements, non-goals, acceptance criteria, verification, and review focus.
- Fullstack implementation starts only after the spec is concrete.
- Independent review happens after implementation and verification evidence exist.
- G胖 owns final synthesis and acceptance.

Default flow:

1. Spec.
2. Fullstack implementation.
3. Local verification.
4. Independent code review.
5. Acceptance or fix loop.

Do not let a broad research idea turn into code without an explicit build target.
